## üìã Screenshot Guidelines

### Recommended Screenshots
1. **Main Interface** - Show the complete application
2. **All 4 Visualization Modes** - One screenshot for each mode
3. **Equalizer Interface** - Show the professional equalizer
4. **Mobile View** - Demonstrate responsive design
5. **Playlist Interface** - Show audio file management
6. **Advanced Controls** - Show advanced features

### Technical Specifications
- **Format**: PNG (preferred) or JPG
- **Resolution**: 1920x1080 or higher
- **Quality**: High quality, clear and sharp
- **File Size**: Under 5MB each
- **Naming**: Use descriptive names with hyphens

### Content Guidelines
- **Clear Interface**: Make sure UI elements are clearly visible
- **Good Lighting**: Avoid dark or blurry screenshots
- **Focused Content**: Show relevant features prominently
- **Consistent Style**: Use similar browser window sizes
- **No Personal Info**: Avoid showing personal information

## üéØ How to Take Screenshots

### Step 1: Start the Project
```bash
# Run the screenshot capture script
./take-screenshots.sh
```

### Step 2: Take Screenshots
1. **Open your browser** to http://localhost:8000
2. **Take screenshots** of different views:
   - Main interface
   - Each visualization mode
   - Equalizer interface
   - Mobile responsive view
   - Playlist interface

### Step 3: Save Screenshots
1. **Save screenshots** to `media/screenshots/` directory
2. **Use descriptive names** like `particle-mode.png`
3. **Ensure good quality** and clear visibility

### Step 4: Upload to GitHub
```bash
# Upload screenshots
./upload-screenshots.sh
```

- **Built-in**: Alt + Print Screen (active window)
